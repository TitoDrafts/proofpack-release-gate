# ProofPack GPT-5.6 Proposal Gate Implementation Plan

> **For Codex:** Execute this plan with the `superpowers:subagent-driven-development` workflow. For every behavior: write the test, observe the expected failure, implement the minimum production change, rerun the focused test, then commit.

**Goal:** Add a real, optional GPT-5.6 Sol proposal workflow whose output is visibly admitted or rejected by ProofPack's existing deterministic compiler, while preserving the no-key/no-network default.

**Architecture:** A repo-scoped Codex skill produces a closed JSON proposal from one synthetic operator email. A pure proposal module validates packet binding, exact anchors, and a two-slot allowlist; it can materialize only the already supported fixed traveler receipt. The web app reviews a checked-in artifact generated by the live command, applies only admitted bindings, and displays the causal diff. The model never sets a claim status or release decision.

**Stack:** TypeScript 5.9, Node 22, `node:test`, React 19, Next/vinext, Codex CLI 0.144.6.
